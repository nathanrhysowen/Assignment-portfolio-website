Type task

Slide 1: Axial

An axial design is a design that centers around an axis. It doesn't matter if it is symmetrical or not, as long as the design is centered around an axis, it is an axial design. For example, in my design, I placed a triangle on the left side of the canvas and used the diagonal line through the center of the canvas as the axis that the text is placed on. For all of my designs I used the font Futura PT. I chose this font from the others because I liked the round edges and felt like it fit the look of the website I would be eventually placing it on. This is also why I chose to make all of the designs blue, since I had planned that my website was going to have a blue colour palette.

Slide 2: Radial

A radial design is layed out in a way where there is a central point with with text radiating from that point. For example, in my design I placed a semi circle to the right of the page and made the text radiate from the left of it. With any design it is important to think about the hierarchy of the text, which piece of text is the most and least important. I placed the title in the centre of the image and made it the biggest because I believe the users eyes will be drawn to the center of the image first.

Slide 3: Dilatational

A dilatational design is a design that is set along circular paths. Thankfully in Illustrator there is a "type along a path" tool that helped a bunch with this layout. For my design, I placed 6 blue circles on the canvas and wrapped the text around them. I placed the title in the top left of the canvas since, atleast in the west, we are engrained to read from left to right and since i couldn't dictate the text hierarchy based on size since the largest piece of text needed the largest circle.

Slide 4: Random

A random design is exactly that, random. It is not dictated by any specific layout. In my opinion, this design was the hardest out of them all because it's a difficult task to make something so random and yet still look appealing. For my design I enclosed the title inbetween two lines and placed the rest of the text at the bottom and top of the page. Each piece of text is a different size to accentuate the "randomness" of the image.

Slide 5: Grid

A grid design is a design where the layout is specified by a grid. For mine, I used the grid layout on Illustrator by making sure that every single piece of text was inside one of the grid boxes. The title lies ontop of one of the grid lines and the text is placed below that line. This layout seems to be one of the most popular layouts, particularly for websites. Most likely beacuse of it's easy to read and clean design. I definietly found this layout the easiest as the fact that I had to restrict myself to using grids meant that I didnt have to think too much about how my text was going to look.

Slide 6: Modular

A modular design is characterised by repeating structures or images that seperate or contain the text. For example, in my design I used the repeating structure of blue squares that contain each piece of text. I made sure that each square was exactly the same size and that the spaces between each square are exactly the same so that it is all uniform.

Slide 7: Transitional

The transitional design was, at first, hard to wrap my head around as it was difficult to find a definition. Type365.com defines it as " they look like the cross section earthâ€“with layers of sediment and stone." Once i read this definition I then understood the concept. In my design, I warped each piece of text to make it seem as if it was layered, and I created a wavy blue blackground to enforce that idea.

Slide 8: Bilateral

Bilateral designs are centered designs that are symetrical. For my design, I placed 3 lines vertically down the page, each seperated with the same amount of space, and placed the text down the center of the page going through the lines.

Strategy and Scope

The fundemental goal of this website is to display all of my tasks completed in this module in a way that is easy to read and nice to look at. In addition, another goal is to fulfill the assignment requirements to a standard that I, myself, am happy with. The users of the website are teachers, possibly parents/grandparents, friends and me. When designing the website it is important to note the different users and how each user will navigate the page and react. For example, I will purposely make the navigation menu easy to find at the top of the page, so that users who are not familiar with navigating many websites (my grandparents) will hopefully instinctively click on those buttons. The website is ocean themed and will contain the CARP task, the coding tasks, the type task and UX task all on one page so that when the user is scrolling down it looks like the user is getting deeper and deeper into the ocean.

Reflection

This website was a blast to make, learning all of the techniques I needed was very interesting and I enjoyed implementing them onto the site. If I could do this again later on when my knowledge of CSS and HTMl is broader I would make the website feel more fluid and interactive. The image sliders were by far the hardest part of the whole project, for some reason, it was difficult to get them to be the proper width and height while also making sure they accurately slide. This website also has issues of scalability where texts and images will display in different positions depending on the screen size. As scalability is not the focus of this module, I decided to leave it as is but for the next assignments it is definitely something I will need to worry about. Another thing I found quite difficult was the animations for the title and pictures at the top of the page. I had to find a way to make the images move as the user scrolled down the page and the best solution I could find was to use Javascript. Thankfully, the creative coding module helped a lot with this, as now I had a basic knowledge of the programming language. I found a tutorial on youtube (which will be linked at the bottom of this paragraph) that helped me write the code out myself and apply it for my own website. Considering at the beginning of the assignment I had no prior knowledge of HTML or CSS, I am proud of the work I've been able to achieve and hope to build upon it in future assignments.
